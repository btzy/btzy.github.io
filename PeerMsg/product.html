The features of PeerMsg



PeerMsg, being a message board, contains some of the most core features that a message board must have. While the features are rather simple, they are extremely useful and important for a person that may be looking for a means of message dissemination for a small organistion.

At the moment, PeerMsg features a login, messaging and local archiving.

Login

In PeerMsg, users can easily create and login to their own personal accounts using a username and a password.

The user will be identified by his username when messages are sent or received.

Messaging

Like any other messaging board, PeerMsg allows users to send messages to one another.

At the moment, a user can send a message to another user simply by inputting a title, the recipient's login username and the message. After which, the message can be sent with the click of a button.

Users can also send a message to multiple recipients at once by inputting several usernames, separated by commas. A user can type ALL in the recipients field to send the message to all users on the network.

Users that are offline at the time of the message being sent will still receive the message so there is no worry that a message will be lost.

Local Archiving

Unlike traditional message boards, where archiving is done server-side, PeerMsg allows the user is done client-side.

With more power over archiving, users no longer need to be restricted to an archive that is shared by everyone using the message board, which may be messy or irrelevant to the specific user. PeerMsg gives users the power to create their own personal archives to store the messages they deem important.

Furthermore, since these archives are stored on the user's computer, no active network or internet connection is required for users to view these archives, meaning that they are accesible offline.

Be it adding or removing a message from an archive, the user simply needs to click the relevant button and the client does the rest.
