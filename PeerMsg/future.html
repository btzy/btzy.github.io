Future Development


Although the product for PeerMsg is a simple messaging board, the most important and useful product of the project is the network code. With the solid foundation of a network code, it is possible to make improvements to the message board.

Client-Side changes

We have a few client-side changes planned for implementation.


The first client-side change that we are working on is a search function. The search function will enables users to filter through a large number of messages to find the message that they are looking for.

The search function will mainly be used to filter messages by their title or content. Users will be able to search for keywords to find the message(s) that they are looking for.

There is already some progress made on the feature but it has not been implemented into the client yet.


The second client-side change that we are trying to complete is an updated login box. The current login box has a rather basic design and we are looking to improve it so that it can accomodate more features such as password input which would allow users to create private accounts.

Progress is being made rapidly for the new login box.

Security

We will also be looking into a some improvements into the network, although it is not a priority at the moment.

Specifically, we are looking to make the transferring of messages more secure so that users will feel confident using PeerMsg even for their most private messages.

It is also very important that the password verification process is extremely secure so that other users on the network are not able to retrieve the password of another user while he attempts to login.
